# 未来のモビリティについての調査
- 方針(論文の様式に沿って調査していけばいい気がする)
  - 現在のモビリティはどのようなものが登場しているのか？(背景)
  - 現在のモビリティの問題点は何か？(背景)
  - その問題点を踏まえて世界全体はどのようなモビリティを目指しているのか？(背景)
  - その問題点に対する解決策はどのようなものが出ているか？(従来技術、先行研究)
  - 現在解決できていない問題点は何か？(目的)

### 過去の歴史
- 馬車・人間の時代
- 初めての車
  - 蒸気で走る車
  - 「ニコラ・ジョセフ・キュニョー」が開発
  - 軍隊で使われる大砲運搬のために造られた重くて大きな車体
  - スピードは10km/h以下だった
  - 世界初の交通事故を起こした自動車
- 新方式の蒸気機関
  - ワットが開発
  - 既得権を守ろうとする馬車業者などからの抵抗は大きかった
- 電気式四輪トラック
  - 1873年に発明
- ジャメ・コンタント号
  - 1899年に発明
  - 電気自動車
  - 史上初の時速100㎞超えを達成
- ガソリン自動車の誕生
  - ドイツ人のゴットリープ・ダイムラーは4ストロークエンジンを開発
  - 1885年に木製の二輪車にエンジンを載せて試走に成功し、翌1886年に四輪車を開発
  - 同じ1886年、同じくドイツ人のカール・ベンツがガソリンエンジンの三輪車を完成させて実際に販売
- FR方式自動車
  - 「システム・パナール」と呼ばれる、現在の自動車技術の基礎を築いた
  - 同時期にソリッドタイヤから空気タイヤができた
- 自動車の量産化
  - T型フォード
  - 1912年キャデラックに初めて採用されたセルフ・スターターの発明が大衆化される
- 日本における自動車の芽吹き
  - 乗用車の第1号「ダット」を完成
- 第二次世界大戦と自動車
  - 1938年にドイツの国民車として、フォルクスワーゲン・ビートルが誕生する。このクルマはポルシェ社の創設者フェルディナント・ポルシェの手により開発
- 色々な自動車の普及
  - トヨタカローラなど色々
- 新しい時代の車
  - パーソナルモビリティやIT技術と連携して、自動ブレーキや自動運転など

### 未来の車
- ハイブリッドカーや電気自動車、水素自動車といったさまざまなエコカーが開発されたことで、従来のエンジン車は2040年ごろに衰退するだろうと予測
- 2030年の販売台数予想では、依然として9割をガソリン車が占め、残り1割がエコカーという見方もある
- エコカーの普及が思うように進まないのは、車両価格の高さと、燃料供給などのインフラ整備の遅れを指摘する声もある
- アメリカのシェールガス採掘拡大で石油価格が下落していることもあり、ガソリン車の人気はまだ続くだろうとも言われている
- パーソナルモビリティという「車」と「歩行者」の間を補完する乗り物
- 自動運転技術
  - https://www.itworldmedia.com/ja/category/autonomouscar.php
- 安全のためのトヨタの最新技術
  - https://toyota.jp/technology/safety/itsconnect/
- 自動車業界を見るとCASE（コネクテッド、自動運転、シェアード、電動化）による大きな変化が生まれ、クルマの在り方が大きく変わろうとしている

## 業界理解から開発までのフロー
- 「研究」でも「データ分析」でも「アプリ開発」でもどんなことでも仮説思考で進む
- 新手法の提案というよりかは、「巨人の肩の上に立つ」を意識するのがいいかも
- 以下の例では「車のアプリ開発をする業界」ということを設定とする
  - 知りたい業界に置き換えて考えても差し支えない
    - 「心理学」や「詐欺」や「ボードゲーム」や「音声の研究」などに置き換えて考えても良い
### 1. 概念化
- 車 -> 移動するもの
  - 「移動するもの」を考えると飛行機で使われるアプリや電車で使われるアプリが車では通用しないのかといったところに繋げることができる可能性がある
- 犯罪心理学 -> 応用心理学 -> 心理学 -> 人の気持ちを読む
- カタン -> ボードゲーム -> ゲーム -> 遊び
  - 「遊びで勝つ方法」、「ゲームで勝つ方法」、「ボードゲームで勝つ方法」、「カタンで勝つ方法」といったことを調べるという方針が立てられる
### 2. 調査: 知識を蓄える段階
1. 業界の歴史「過去・現在・未来」の調査
  - 過去の調査
    - 車はどのような歴史を経て開発されたのか?
    - 過去の車はどのようなものがあるか？
  - 現在の調査
    - 現在の車はどのようなものがあるのか？
    - 現在の研究段階の車はどのようなものがあるか？
    - 過去の車の課題点をどのように乗り越えてきたのか？
  - 未来の調査
    - 未来はどのような車を目指しているのか？
    - 現在の課題点をどのように乗り越えようとしているのか？
### 3. 仮説を立てる
1. 業界の一般的な問題の調査
  - 現在の車と未来の車の問題点を検索して調べる
2. 一般的な問題点がなぜ起きているのかを考える(水平思考、仮説思考)
  - 水平思考でアイデアをどんどん出していく
    - オズボーンのチェックリストとか...
    - 高齢者の事故ではなくて、「逆に」若者の事故の方が多いのではないか？
  - 「未来へのWhy(なぜやるのか？)」または「過去へのWhy(なぜそうなったのか？)」を使って本質の課題を発見する
    - なぜ車によって地球温暖化が進むのか? -> CO2が原因なのではないか？
    - なぜ死亡事故が多いのか？ -> スピード超過が関係しているのではないか？、高齢者の事故が多いのではないか？、煽り運転が関係しているのではないか？
      - なぜ高齢者の事故が多いのか？ -> 判断能力の欠如が関係している？、ブレーキアクセルの踏み間違えが関係している？
### 4. テーマを確定する: 問題提起・目標の設定
- 「3. 仮説を立てる」で明らかになった問題点をそのまま設定すると良い
- 例: ブレーキアクセルの踏み間違えの防止
### 5. データを収集する
- 例: 年齢別の事故、事故の原因、etc
### 6. 仮説を検証する
- 「5. データを収集する」で得たデータを使って「2. 仮説を立てる」で立てた仮説を検証していく
### 7. 新たな仮説を立てて繰り返す
- 「6. 仮説を検証する」で明らかになった事実から、新たな仮説を立てて「2~4」を繰り返していく
### 8. モノを作る
1. 従来手法の調査
  - 設定した課題点に対して、すでに作成されているアプリは無いか調べる
2. 提案手法の開発
  - 従来手法のアプリにプラスの機能をつけてパワーアップさせるパターン
    - 水平思考のフレームワークを使ってアイデア出しを行う
  - 新しいアイデアからアプリを開発するパターン
    - 水平思考のフレームワークを使ってアイデア出しを行う
### 9. 実験
- 「8. モノを作る」で開発したアプリの効果を検証する
  - 実際に使ってみた時の感想
  - アプリのレスポンス
  - etc
### 10. 考察
- どのような効果がありそうか
- どこを改善できると良さそうか
- etc
### 11. 結果をまとめ、発表する
- 論理的に発表できるように
- パワポのフレームワークを見ながらわかりやすく
- PREPに沿うなど
